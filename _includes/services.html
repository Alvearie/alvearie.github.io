Alvearie is made up of many individual services and components that can be used together to build a health data pipeline. Below is a starter list of those services, which will be growing over time.

Linux for Health Connect LinuxForHealth (LFH) is a distributed processing network operating system which allows edge devices (IoT, workstations, app servers) to connect directly to health care transaction systems. The processing model abstracts the need for intermediary third-party organizations, resulting in a developer-extensible trust protocol. Go to GitHub repository Documentation
IBM FHIR Server IBM's implementation of the HL7 FHIR specification simplifies data management by allowing you to exchange disparate datasets between applications that were previously incompatible. Go to GitHub repository
Health Record Ingestion A "front door" for data integrators to send healthcare data into the IBM cloud, while supporting both batch processing and data streaming workflows. Go to GitHub repository
HRI Pipeline Adapters NiFi flows that consume data from the Health Record Ingestion service Go to GitHub repository
Data De-Identification A wide range of de-identification capabilities designed to support GDPR, HIPAA, CCPA and other privacy frameworks allowing organizations to meet their regulatory and privacy requirements. Go to GitHub repository
Quality Measure and Cohort Service Service to evaluate quality measure and cohort specifications against a target patient data set. Go to GitHub repository
Alvearie FHIR Implementation Guide A FHIR Implementation Guide that extends the US Core IG. Go to GitHub repository
HL7 Ingestion Service A service that transforms HL7 messages into a bundle FHIR resource. Go to GitHub repository
HL7 to FHIR Converter A java-based library that enables conversion of HL7 v2 messages to FHIR resources Go to GitHub repository
